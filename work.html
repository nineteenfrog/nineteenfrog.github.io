Procedural Animation Demo

I wanted to explore the concept of procedural animation for small gestures like foot placement and also reacting to obstacles in the environment. I used Unity's Animaiton Rigging package to create some controllers and constraints in real time. Then I created this small demo in which the character touches the wall as he walks by and also adjusts the foot placement based on an obstacle. This was a mere proof of concept and I want to maybe implement a more robust demo. I might even switch into Unreal Engine. Currently exploring where I could take this but this was a fun and interesting project I have been working on.

Bird Rig

The project started with the idea of rigging a bird and doing a physics based feather flocking animation. I had recently lost my bird pet named Loki (he escaped :c) so I decided to do use him as reference for my bird project. I researched what species they were and used pictures to see anatomy and model in Maya. I did a low poly model and rigged all the body as well as each individual feather. Then I added some dynamic curves for the feather joints to connect to. I animated a wing flapping animation and the feathers moved dynamically. Then I created a fur system using xGen for the feathers. I turned those feathers into polygons and then did a skin bind to the rig. Once this was done, I brought it to Substance Painter where I gave it proper textures based on its species (Agapornis personatus) and finally created a flocking animation using instances of the bird. Note: Model wasn't optimized for a runtime application because I was just going to render a video of the flocking animation. For game assets the ideal scenario would have been to use alpha cards.

Robot Rig

The project started with me creating the mesh. Since it was a robotic character I modeled individual pieces and then put them together using Maya. Also using Maya, I created a rig that has an IK/FK switch. Once the Rig was done I took the model to Substance Painter. I realized I hadn't done the UVs for the model but since I was using many repeated pieces in the model, I UV unwrapped one of each piece and transfered the UVs to save time. Then I took it back to Substance where I textured it. Once textured, I exported an FBX from Maya and imported it to Houdini where I created a crowd simulation. Later I used the asset in one of my game projects Into the Rift. I helped create an animation blueprint for the character and one of my teamates programmed simple AI logic so that the robot would follow the player.

Ball Rig Project

Ball Rig - This rig focused on highlighting squashing for animating a bounce. As seen on the maya outliner, a squash handler was added to the ball geometry. Then the master control was given a channel so that an animator can adjust and key the value of the squash effect. After creating a animation, I exported as an alembic file into houndini where I instantiated a crowd simulation and alternated the animation frame as well as the RGB values of every ball. The render of this can be seen in my demo reel in the home page.
